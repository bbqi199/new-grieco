## 订单生成Excel下载（邮件方案已废弃）
- 原计划：Outlook SMTP 发邮件 → 失败（微软2024年全面禁用SMTP基础认证，应用密码也不行）
- 最终方案：提交订单时前端用 SheetJS 直接生成 Excel 并下载到本地
- 文件名格式：`订货单_{订单号}.xlsx`，包含商品编号/名称/规格/订货数量四列，带列宽设置
- server.py 已精简，移除所有邮件相关代码（smtplib/openpyxl/EMAIL_CONFIG）

## server.py 彻底修复 CLOSE_WAIT 堆积（第三次）
- 根本原因：HTTP/1.1 keep-alive 让浏览器复用连接，服务端没主动关闭导致 CLOSE_WAIT 堆积
- 修复方案：`protocol_version = "HTTP/1.0"` 禁用 keep-alive + 所有响应加 `Connection: close` + `get_request` 里 `SO_LINGER=0` 强制 RST
- 彻底解决，连接使用后立即释放，不再堆积

## quickAddCart 加入数量按规格自动计算
- 修改 `quickAddCart` 函数：不再固定加1，而是从 `specs[0]` 中用正则 `/\d+/g` 提取最后一个数字作为加入数量
- 例：`'6包/提'`→6，`'550ml×24'`→24，`'10片装'`→10，无数字时默认1
- Toast 提示也同步显示实际加入数量，如"+6 已加入购物车"
